# かんたん寄せ書き 具体化ドキュメント

## サービス概要

退職・送別・誕生日・結婚祝いなどの寄せ書きをオンラインで集めて、きれいなレイアウトで贈れるWebサービス。幹事がURLを共有するだけで、メンバー全員がメッセージを書き込める。完成したらPDF・画像として出力でき、希望者には実物の色紙として印刷・郵送するサービスも提供する。

---

## 想定される使われ方のシナリオ

### シナリオ1: 退職する上司への寄せ書き
- 幹事の田中さんが「山田部長 退職おめでとうございます」という寄せ書きを作成
- 締切を退職日の3日前に設定
- 部署のSlackやメールでURLを共有
- 在宅勤務メンバーや出張中のメンバーも含めて20人がメッセージを記入
- 締切後、PDFをダウンロードして印刷、または実物の色紙として印刷代行サービスを利用して郵送

### シナリオ2: 友人の結婚祝い
- 新婦の親友が幹事となり「○○ちゃん結婚おめでとう」を作成
- 学生時代の友人15人にURLを送信
- 写真付きメッセージや動画メッセージも投稿可能
- 結婚式の前日に印刷物として届くよう手配

### シナリオ3: 遠方の祖父母への誕生日メッセージ
- 孫4人が祖母の80歳の誕生日に向けて寄せ書きを作成
- 全国に散らばる家族(子・孫・ひ孫合わせて15人)からメッセージを集める
- 印刷代行で実物の色紙として祖母の家に届ける

### シナリオ4: 部活の卒業生送り
- 高校の部活で卒業生5人への寄せ書きを後輩30人で作成
- 1人ずつ別々の寄せ書きを並行して作成可能
- 卒業式当日に紙の色紙として渡す

### シナリオ5: 育休に入る同僚への送別
- リモートワーク中心の職場で、対面で寄せ書きを回せない
- 全国の支社メンバー含めてオンラインで集める

## 技術構成

### フロントエンド
- **Next.js (App Router)** + TypeScript
- **Tailwind CSS**
- **shadcn/ui**
- **dnd-kit**(メッセージカードのドラッグ&ドロップによる並び替え用)
- ホスティング: **Vercel**

### バックエンド
- Next.js の **API Routes**
- 認証なし(URL=アクセス権)、ただし幹事用URLと参加者用URLを分離

### データベース・ストレージ
- **Supabase** (PostgreSQL + Storage)
  - 写真アップロード用にSupabase Storage を活用
- 寄せ書きID は推測困難なランダム文字列

### PDF生成
- **react-pdf** または **Puppeteer** によるサーバーサイド PDF 生成
- レイアウトはメッセージ数に応じて自動調整(3〜30件まで対応)

### 印刷代行
- **印刷会社のAPI連携**(プリントパック、ラクスルなどとの連携を検討)
- または初期は手動運用(注文を受けたら自社で印刷・発送)

### 決済
- **Stripe** または **Square**
- クレジットカード・コンビニ払い対応

### データモデル(簡略)
```
boards
  - id (string, primary key)
  - admin_token (string) // 幹事用URLに含める識別子
  - title (string)
  - recipient_name (string)
  - deadline (date)
  - host_name (string)
  - template (string)
  - is_locked (boolean) // 締切後はロック
  - created_at (datetime)
  - expires_at (datetime) // 締切日 + 90日で自動削除

messages
  - id (uuid)
  - board_id (foreign key)
  - author_name (string)
  - content (text)
  - card_color (string)
  - icon (string, nullable)
  - photo_url (string, nullable)
  - position (integer) // 表示順
  - created_at (datetime)

print_orders
  - id (uuid)
  - board_id (foreign key)
  - size (string)
  - finishing (string)
  - quantity (integer)
  - shipping_name (string)
  - shipping_address (string)
  - shipping_phone (string)
  - desired_delivery_date (date)
  - amount (integer)
  - status (string) // pending, paid, printed, shipped, delivered
  - stripe_payment_id (string)
  - created_at (datetime)
```

### その他
- セキュリティ: 幹事用と参加者用の URL を分離し、削除・ロックは幹事のみ可能
- 写真の自動リサイズ(アップロード時)
- 不適切画像検出(将来的に Cloud Vision API などを検討)

- **`docs/print-operation.md`** に印刷代行の運用マニュアルを残し、初期は手動運用、後に自動化する移行をスムーズに

## MVP の最小機能定義

### MVP に含める機能
1. **寄せ書き作成**(タイトル・贈る相手・締切・幹事名)
2. **URL共有**(幹事用URL・参加者用URLの2種類)
3. **メッセージ投稿**(名前・本文・カードの色選択)
4. **メッセージ一覧表示**(グリッドレイアウト)
5. **PDFダウンロード**(基本レイアウト1種類のみ)
6. **メッセージの削除**(自分が書いたもの・幹事は全件)

### MVP に含めない機能(あえて削る)
- 印刷代行サービス(フェーズ2以降)
- デザインテンプレート選択(まずは1種類のみ)
- イラスト・絵文字の挿入
- 写真添付
- 動画メッセージ
- メッセージの並び替え(自動レイアウトのみ)
- 締切後の自動ロック

### MVP のゴール
- 開発期間: **4〜5週間**(PDF生成の実装に時間がかかる想定)
- 「実際の寄せ書きシーンで使われるか」を検証
- 知人の退職や誕生日のタイミングで実際に使ってもらう

## 競合分析と差別化

### 主な競合
| サービス | 特徴 | 弱点 |
|---|---|---|
| yosetti(ヨセッティ) | 業界先発、デザイン豊富 | 有料前提、1人550円〜 |
| 寄せ書きMAGNET | テンプレート豊富 | デザインが古め |
| sugar(シュガー) | 動画寄せ書き特化 | 動画のみで使い方が限定的 |
| Googleドライブで自作 | 無料 | デザインを一から作る必要、参加が面倒 |
| 紙の色紙(伝統的方法) | 実物が残る | 遠隔地メンバーが参加できない |

### かんたん寄せ書きの差別化
- **基本機能は完全無料**(競合は有料前提が多い)
- **登録不要で参加可能**(参加者のハードルが極小)
- **印刷代行で実物も得られる**(オンライン × オフラインのハイブリッド)
- **シンプルさと美しさの両立**(機能を絞り込みつつデザインは妥協しない)

## 収益モデル(他案より明確)

このサービスは**3つの収益源**を持つ:

1. **印刷代行**(主要収益)
   - 1件あたり ¥2,500〜¥3,500
   - 想定粗利率: 50〜60%

2. **有料プラン**(月額)
   - 月 500 円程度
   - 動画メッセージ無制限・参加人数無制限・広告非表示
   - 主に頻繁に使う法人ユーザー向け

3. **ギフト連携**(アフィリエイト)
   - 花束・スイーツ・カタログギフトなどとセット販売
   - 提携先からの送客手数料
